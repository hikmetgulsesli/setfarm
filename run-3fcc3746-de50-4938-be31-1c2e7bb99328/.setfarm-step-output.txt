## 13. Mimari Kararlar

### DECISION: Next.js App Router + Static Export
**ALTERNATIVES_CONSIDERED:** Vite React SPA, plain HTML/CSS
**RATIONALE:** Next.js App Router provides SSR/SSG flexibility, file-based routing, built-in image optimization, and excellent developer experience. For a portfolio site, we need SSR for SEO but can prerender most pages statically. App Router's server components reduce client-side JS bundle.
**TRADE_OFFS:** Larger framework footprint than Vite, steeper learning curve for contributors unfamiliar with Next.js.
**PIVOT_POINT:** If build times become prohibitive, switch to pure static export mode.

### DECISION: shadcn/ui + Tailwind CSS
**ALTERNATIVES_CONSIDERED:** Chakra UI, MUI, plain CSS modules
**RATIONALE:** shadcn/ui provides accessible, customizable components built on Radix primitives. Tailwind enables rapid styling with consistent design tokens. No runtime CSS-in-JS overhead.
**TRADE_OFFS:** shadcn/ui requires manual setup and copy-paste rather than npm install.
**PIVOT_POINT:** If design requirements diverge significantly from shadcn patterns, evaluate pure Radix + custom styles.

### DECISION: Framer Motion for Animations
**ALTERNATIVES_CONSIDERED:** CSS animations, React Spring, Motion One
**RATIONALE:** Framer Motion has best-in-class React integration, declarative API, layout animations, and gesture support. Wide adoption means good documentation and community support.
**TRADE_OFFS:** Adds ~30KB to bundle. Some complex animations require client components.
**PIVOT_POINT:** If performance budget is tight, fall back to CSS animations for simple transitions.

### DECISION: MDX for Content
**ALTERNATIVES_CONSIDERED:** CMS (Contentful/Sanity), plain Markdown
**RATIONALE:** MDX enables writing content in Markdown while embedding React components inline. No external CMS dependency, content lives in the repo (version control, easy deploys).
**TRADE_OFFS:** Build time scales with content volume. No visual CMS editor for non-technical authors.
**PIVOT_POINT:** If content editing by non-technical team members becomes a priority, migrate to Sanity or Contentful.
